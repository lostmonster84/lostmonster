# Pre-Design Quality Checklist - Lost Monster Website

> **Customized for Lost Monster website redesign**
>
> **Purpose:** Ensure design variations meet Lost Monster's quality standards before implementation
>
> **Minimum Threshold:** 80/100 points

---

## üéØ Purpose

**Problem:** Design principles get forgotten during implementation, leading to:
- Inconsistent brand execution
- Design patterns that don't match Lost Monster's positioning
- Quality drift over time
- Website that doesn't demonstrate Lost Monster's methodology

**Solution:** Mandatory checklist ensures design principles are verified BEFORE choosing final design variation.

## ‚úÖ The Checklist Structure

### Category 1: Framework Adherence (25 points)

**Why this matters:** The website must demonstrate Lost Monster's framework-driven approach. If we don't follow our own principles, we lose credibility.

- [ ] **CODA planning documented** (5 points) - CODA plan exists and is referenced
- [ ] **Design variations created** (5 points) - 5 distinct variations built and visible
- [ ] **Quality checklist applied** (5 points) - This checklist used to score variations
- [ ] **Domain knowledge reflected** (5 points) - Design decisions align with domain knowledge
- [ ] **Systematic approach visible** (5 points) - Process transparency, methodology showcased

**Why this matters:**
- Credibility: Website demonstrates Lost Monster's methodology
- Differentiation: Shows systematic approach vs typical agencies
- Sales tool: Proves we practice what we preach
- Quality assurance: Ensures we meet our own standards

**Red flags:**
- ‚ùå No CODA plan documented
- ‚ùå Only one design approach (no variations)
- ‚ùå Quality not scored
- ‚ùå Generic design (doesn't reflect domain knowledge)
- ‚ùå Ad-hoc approach (no systematic process visible)

**Good examples:**
- ‚úÖ CODA plan linked in README
- ‚úÖ Demo pages showing 5 variations
- ‚úÖ Quality scores documented
- ‚úÖ Design reflects premium, systematic positioning
- ‚úÖ Process transparency (show methodology)

---

### Category 2: Design System Consistency (20 points)

**Why this matters:** Consistent design system demonstrates professionalism and systematic approach. Inconsistencies undermine credibility.

- [ ] **Color palette consistent** (4 points) - Colors used consistently across all pages
- [ ] **Typography hierarchy clear** (4 points) - Heading/body fonts, sizes, weights consistent
- [ ] **Spacing system followed** (4 points) - 8px base unit system used consistently
- [ ] **Component patterns consistent** (4 points) - Buttons, cards, sections follow same patterns
- [ ] **Design tokens used** (4 points) - Tailwind config tokens used, not hardcoded values

**Why this matters:**
- Professionalism: Consistent design = professional appearance
- Maintainability: Design system makes updates easier
- Quality signal: Attention to detail visible
- User experience: Consistency improves usability

**Red flags:**
- ‚ùå Colors vary randomly across pages
- ‚ùå Typography sizes inconsistent
- ‚ùå Spacing feels arbitrary
- ‚ùå Components look different on different pages
- ‚ùå Hardcoded colors/values instead of tokens

**Good examples:**
- ‚úÖ Consistent primary/secondary/accent colors
- ‚úÖ Clear typography scale (xs, sm, base, lg, xl, etc.)
- ‚úÖ Spacing follows 8px grid
- ‚úÖ Components reusable and consistent
- ‚úÖ All values from Tailwind config

**Good examples:**

### Category 4: Technical Quality (20 points)

**Why this matters:** Technical quality demonstrates Lost Monster's competence. Poor performance or accessibility undermines credibility.

- [ ] **Performance optimized** (4 points) - Core Web Vitals pass, Lighthouse 90+
- [ ] **Accessibility compliant** (4 points) - WCAG 2.1 AA minimum

- [ ] **Code quality** (4 points) - TypeScript, linted, documented
- [ ] **Mobile-first** (4 points) - Responsive, 60%+ mobile traffic considered

**Why this matters:**
- Credibility: Technical quality = competence
- User experience: Fast, accessible site = better UX
- SEO: Proper SEO = more traffic
- Maintainability: Clean code = easier updates

**Red flags:**
- ‚ùå Slow load times
- ‚ùå Accessibility violations
- ‚ùå Poor SEO (no meta tags, etc.)
- ‚ùå Sloppy code
- ‚ùå Mobile experience broken

**Good examples:**
- ‚úÖ Lighthouse scores 90+ all categories
- ‚úÖ WCAG 2.1 AA compliant
- ‚úÖ Proper semantic HTML, meta tags
- ‚úÖ Clean TypeScript code
- ‚úÖ Mobile-first responsive design

## üõë STOP Checkpoint

**Before choosing a design variation, ask yourself:**

1. **Did I check EVERY item above?**
2. **Can I honestly check ALL boxes?**
3. **If this were deployed tomorrow, would it demonstrate Lost Monster's methodology?**
4. **Does it match Lost Monster's positioning (premium, systematic, results-focused)?**
5. **Would potential clients trust Lost Monster based on this design?**
6. **Does it score 80+/100?**

**If you answer "NO" or "UNSURE" to ANY question:**
- **STOP**
- Identify which criteria are failing
- Make targeted improvements
- Re-score until 80+ achieved

**Only proceed when ALL criteria are met and score is 80+/100.**
