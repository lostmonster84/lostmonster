## Contributing to Documentation

Good documentation is crucial. When adding new components, integrations, or patterns:

1. Create or update relevant documentation
2. Include code examples
3. Document edge cases and gotchas
4. Add screenshots/diagrams where helpful
5. Link to related resources

## Documentation Standards

- Use clear, concise language
- Include practical examples
- Keep it up-to-date with code changes
- Cross-reference related documentation
- Use consistent formatting and structure
